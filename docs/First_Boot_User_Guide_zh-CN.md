# 💻 欢迎使用新系统 - 首次开机与配置指南

尊敬的用户，您好！
本设备已为您预装了完整定制的 Windows 11 操作系统及相关后端组件环境。为了确保所有 AI 服务和容器环境能够正确初始化，**在您首次开机时，系统会进行一次自动配置**。

请您仔细阅读本指南，按照提示完成首次开机流程。

---

## ⏳ 第一阶段：基础系统配置（约 2~3 分钟）

1. **按下电源键开机**：电脑启动后，系统会自动跳过繁琐的隐私协议和强制联网页面，直接进入账户设置阶段。
2. **创建本地账户**：
   - 屏幕上会提示您“谁将会使用这台电脑？”。
   - 请在此处输入您想要的用户名（建议使用英文字母，避免特殊符号）。
   - 设置并确认您的登录密码。
3. **等待进入桌面**：设置完账户和三个安全问题后，系统会显示“正在为您准备一切”的过渡动画。请耐心等待，直到进入熟悉的 Windows 11 桌面。

---

## ⚠️ 第二阶段：全自动部署安装（进入桌面后，极其重要！）

**【核心警告】：当您首次看见 Windows 桌面后，请不要立刻开始日常使用，也不要重启电脑！系统后台正在为您配置核心业务环境（如 Docker 等环境引擎和镜像数据）。**
*💡 提示：同型号的机器在出厂前均已经过严格的部署测试。请在第一次启动时务必耐心等待自动配置完成，切勿强行中断操作，以避免环境初始化故障和不必要的系统重装。*

1. **底层引擎启动**：
   系统会在后台自动唤醒 Docker Desktop 等容器引擎。您可能会在右下角任务栏看到 Docker 的小系统图标出现并闪烁（正在启动）。
2. **弹出黑色配置窗口**：
   几分钟内，屏幕上会自动弹出一个或多个 **黑色的命令行配置窗口**（CMD 窗口）进程。
   - 画面上会显示 `loading images...` 或 `installing service...` 等字符。
   - 这是系统正在为您加载预置的大型数据镜像。
   - 🚫 **绝对不要手动点击右上角的“X”关闭这些黑色窗口！** 如果强制关闭，可能会导致应用商店或业务组件损坏。
3. **耐心等待**：
   解压和载入镜像需要大量的磁盘读写（根据设备性能，可能需要 5 到 15 分钟）。如果窗口停留在某一行不动，这是正常现象，请让它保持在屏幕上运行。

---

## 🎉 第三阶段：部署完成与信息留存

当所有后台任务和命令行窗口跑完后，黑框会自动消失，并向您展示部署结果：

1. **配置凭证弹窗（AppStore / 1Panel 等）**：
   如果您的设备预装了相关应用商城或后端管理面板，屏幕中央会弹出一个提示窗口，里面包含了**默认的管理员账号、密码以及访问地址**。
   📝 **请务必使用手机拍照或截图保存这些重要凭证信息！**
2. **成功提示弹窗（CIKE 或其他业务）**：
   最后，您会看到一个类似于“安装成功 / Success”的确认弹窗。
3. **大功告成**：
   当确认所有弹窗都已展现，且不再有黑色命令窗口运行后，意味着整个极其复杂的出厂环境已全部署妥当。
   ✔️ **现在，您可以自由、正常地使用这台电脑了。你可以安全地重启机器或开始连接 WiFi 上网。**

### ❓ 常见问题解答 (FAQ)

**Q：我不小心随手关掉了那个黑色的命令行窗口，中断了安装怎么办？**
A：强制中断会导致环境加载不完整。但请放心，系统具有自动重试机制。如果因为误操作中断了安装，您可以通过以下两种方式恢复：
1. **直接重启电脑**（推荐）：再次开机并登录账户后，系统会自动检测未完成的任务，并重新弹出黑框继续安装流程。
2. **手动执行**：如果您希望立刻恢复，可以重新注销当前账户并再次登录，或者按下 `Win + R` 键打开“运行”对话框，输入 `C:\ProgramData\FirstBoot\` 并回车，随后双击运行文件夹中的 `firstboot-launcher.vbs` 脚本来手动启动部署流程。

**Q：启动后十多分钟了，黑框一直没出来，可以直接用吗？**
A：如果右下角 Docker 图标迟迟未变绿启动成功，或者超过 15 分钟没有任何反应，可能是底层引擎启动超时被挂起。您可以尝试重启一次电脑，系统在第二次登录时会再次尝试拉起未完的自动部署任务。

**Q：下次再重启电脑，还会弹出这些黑框和等这么久吗？**
A：**不会。** 只要首次开机的自动化流程完整、成功地结束，系统就会写入一个“完成标记”。以后的每一次开机和登录，都会像普通电脑一样秒进桌面，不会再有任何打扰。
